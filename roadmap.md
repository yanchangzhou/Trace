📄 Trace 项目状态与路线图 (Roadmap)
1. 项目简介
Trace 是一款基于 Tauri (Rust) + React (TypeScript) 开发的极简知识管理工具。它旨在为研究者和创作者提供一个干净、高效的沉浸式工作环境。

设计哲学：克制、温暖（暖奶油色调）、极致性能。

核心能力：全格式文件预览、深度语义编辑器、个性化 AI 辅助。

2. 当前完成进度 (Milestones)
目前项目已完成基础架构搭建，并解决了几个关键的交互痛点：

[已完成] 视口架构与布局安全：

实现了三栏式布局（列表 - 预览 - 编辑）。

建立了 300px 编辑器安全区，确保预览框无论如何拉伸都不会完全盖死编辑器。

[已完成] 文档预览引擎优化：

PDF/Docx 自动适配：解决了右侧内容截断问题，默认实现 Fit-to-Width。

零闪烁缩放：通过“静态防抖渲染”技术，彻底消除了调整大小时的画面撕裂。

后端通路：修复了中文字符路径解析问题，改用二进制流加载，提升了 PDF 加载稳定性。

[已完成] 编辑器初步升华：

Bubble Menu：实现了选中文字后弹出浮动工具栏。

Slash Command (1.0)：搭建了 / 快捷指令的基础框架。

[已完成] 项目瘦身：

将工程体积从 7.1GB 缩减至 ~1.5GB（优化了 Rust 编译配置，清理了缓存）。

3. 接下来的重点攻坚任务
3.1 赋予编辑器“灵性” (The Soul of Editor)
目前的编辑器虽有功能，但还缺乏 Notion 那种“如丝般顺滑”的交互感：

交互补完：完善 / 菜单的键盘导航逻辑与平滑动画。

块级交互：实现 Block-based 交互，支持拖拽调整行块顺序。

视觉微调：优化输入时的文字间距、行高动态调整，以及更加自然的 Focus 状态切换。

3.2 AI API 深度稳定性工程
流式响应 (Streaming)：实现 AI 内容逐字生成，告别长时间等待。

上下文桥接：让 AI 能够“读取”左侧预览框中的文档内容，并在右侧编辑器中生成摘要或对比分析。

断线重连：建立健壮的 API 错误处理机制，确保在弱网环境下也能稳定输出。

3.3 写作风格学习算法 (Style Learner Algorithm)
这是 Trace 区别于普通笔记软件的核心竞争力。

思路：构建一个轻量级的 风格提取器 (Style Extractor)。

步骤：

特征量化：分析用户过去笔记的词频、句式长短比、专业词汇密度。

向量偏移：在使用 LLM 生成时，将提取的特征作为 System Prompt 的“微调偏差”输入。

协同过滤：针对学术写作、日常随笔等不同标签，自动切换相应的语气库。

4. 愿景：从“工具”到“外挂大脑”
Trace 最终将不只是一个显示文件的窗口，而是一个能理解你研究习惯、模拟你表达方式的协作伙伴。
